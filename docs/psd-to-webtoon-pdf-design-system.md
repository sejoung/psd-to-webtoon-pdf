# psd-to-webtoon-pdf — Design System

## 1. Design Philosophy

- Minimal, functional, developer-friendly
- Focus on clarity over decoration
- Optimized for workflow tools (not marketing UI)
- Inspired by VS Code, Linear, Figma

---

## 2. Color System

### Base (Neutral)
- Background: #18181B (zinc-900)
- Surface: #27272A (zinc-800)
- Border: #3F3F46 (zinc-700)
- Text Primary: #FAFAFA
- Text Secondary: #A1A1AA

### Accent
- Primary: #6366F1 (indigo-500)
- Hover: #4F46E5 (indigo-600)

### Semantic
- Success: #10B981 (emerald-500)
- Warning: #F59E0B (amber-500)
- Error: #E11D48 (rose-600)

---

## 3. Typography

- Font: Inter / system-ui
- Headings: semi-bold
- Body: regular
- Small UI text: 12–14px

---

## 4. Spacing & Layout

- Base unit: 8px
- Card padding: 16–24px
- Section spacing: 24–32px

---

## 5. Radius & Shadows

- Border radius: 12px
- Cards: subtle shadow
- Buttons: minimal shadow or none

---

## 6. Components

### Button
- Primary (accent)
- Secondary (neutral)
- Danger (red)

### Card
- Used for sections (file list, options)

### Input
- Text input
- File path display

### Slider
- JPEG quality control

### Progress Bar
- Merge progress

### Toast
- Error / success feedback

---

## 7. Interaction States

- Hover: slightly brighter background
- Active: darker accent
- Disabled: opacity 50%

---

## 8. Icons

- Library: lucide-react
- Style: outline, consistent stroke width

---

## 9. Theme

- Default: Dark mode
- Optional: Light mode (future)

---

## 10. UI Tone

- Clean
- Quiet
- Tool-like
- No unnecessary animations

---

## 11. Summary

This design system prioritizes:
- clarity
- speed
- low cognitive load
- developer-friendly aesthetics
